40  IMBD Movie Reviews

In this tutorial we will work with the IMDB dataset: a set of 50,000 highly polarized reviews from the Internet Movie Database. Theyâ€™re split into 25,000 reviews for training and 25,000 reviews for testing, each set consisting of 50% negative and 50% positive reviews.

The IMDB dataset comes packaged with Keras. It has already been preprocessed: the reviews (sequences of words) have been turned into sequences of integers, where each integer stands for a specific word in a dictionary. This enables us to focus on model building, training, and evaluation.

40.1 Loading the Data Set

The following code will load the dataset (when you run it the first time, about 80 MB of data will be downloaded to your machine).

The argument num_words=10000 means youâ€™ll only keep the top 10,000 most frequently occurring words in the training data. Rare words will be discarded. This allows us to work with vector data of manageable size. If we didnâ€™t set this limit, weâ€™d be working with 88,585 unique words in the training data, which is unnecessarily large. 40.3 Preparing the Data

You canâ€™t directly feed lists of integers into a neural network. They all have different lengths, but a neural network expects to process contiguous batches of data. You have to turn your lists into tensors. In particular we will use multi-hot encoding.

This would mean, for instance, turning the sequence [8, 5] into a 10,000-dimensional vector that would be all 0s except for indices 8 and 5, which would be 1s. Then you could use a Dense layer, capable of handling floating-point vector data, as the first layer in your model.

The following code performs the multi-hot encoding. Notice that we are flattening our text data so that we are losing all sense of ordering of the words.

import numpy as np
def vectorize_sequences(sequences, dimension=10000):
    results = np.zeros((len(sequences), dimension))
    for i, sequence in enumerate(sequences):
        for j in sequence:
            results[i, j] = 1.
    return results
x_train = vectorize_sequences(train_data)
x_test = vectorize_sequences(test_data)

40.4 Building the Model

The input data is vectors, and the labels are scalars (1s and 0s): this is one of the simplest problem setups youâ€™ll ever encounter. A type of model that performs well on such a problem is a plain stack of densely connected (Dense) layers with relu activations.

There are two key architecture decisions to be made about such a stack of Dense layers:

  • How many layers to use
  • How many units to choose for each layer

We will use two hidden layers with 16 units each and a relu activation function. Since we are predicting a single outcome (positive or negative) our output layer will have a single unit and we will use a sigmoid activation because this is a binary classification problem.

from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    layers.Dense(16, activation="relu"),
    layers.Dense(16, activation="relu"),
    layers.Dense(1, activation="sigmoid")
])

Finally, you need to choose a loss function and an optimizer. Because youâ€™re facing a binary classification problem and the output of your model is a probability (you end your model with a single-unit layer with a sigmoid activation), itâ€™s best to use the binary_crossentropy loss. It isnâ€™t the only viable choice: for instance, you could use mean_squared_error. But crossentropy is usually the best choice when youâ€™re dealing with models that output probabilities.

As for the choice of the optimizer, weâ€™ll go with rmsprop, which is a usually a good default choice for virtually any problem.

model.compile(optimizer="rmsprop",
              loss="binary_crossentropy",
              metrics=["accuracy"])

40.5 Creating a Validation Set

A deep learning model should never be evaluated on its training data â€” itâ€™s standard practice to use a validation set to monitor the accuracy of the model during training. Here, weâ€™ll create a validation set by setting apart 10,000 samples from the original training data.

x_val = x_train[:10000]
partial_x_train = x_train[10000:]
y_val = y_train[:10000]
partial_y_train = y_train[10000:]

40.6 Training the Model

We will now train the model for 20 epochs (20 iterations over all samples in the training data) in mini-batches of 512 samples. At the same time, we will monitor loss and accuracy on the 10,000 samples that we set apart. We do so by passing the validation data as the validation_data argument.

40.7 Plotting the Training and Validation Loss

Letâ€™s use matplotlib to plot the training and validation loss side by side as well as the training and validation accuracy. Note that your own results may vary slightly due to a different random initialization of your model.

import matplotlib.pyplot as plt

plt.title("Training and validation accuracy")
plt.xlabel("Epochs")
plt.ylabel("Accuracy")
plt.legend()
plt.show()

As you can see, the training loss decreases with every epoch, and the training accuracy increases with every epoch. Thatâ€™s what you would expect when running gradient- descent optimization â€” the quantity youâ€™re trying to minimize should be less with every iteration. But that isnâ€™t the case for the validation loss and accuracy: they seem to peak at the fourth epoch. What youâ€™re seeing is overfitting: after the fourth epoch, youâ€™re overoptimizing on the training data, and you end up learning representations that are specific to the training data and donâ€™t generalize to data outside of the training set.

In this case, to prevent overfitting, you could stop training after four epochs.
